# Dragon App Docs Template

Copy or generate these documents into each Dragon product repo, then replace the app-specific placeholders:

- `docs/developer-setup.md`
- `docs/data/README.md`
- `docs/testing/test-suite-structure.md`
- `docs/release/release-builds.md`
- `docs/qa/manual-qa-workflow.md`
- `docs/assets/brand-assets.md`
- `docs/troubleshooting/README.md`

The source of truth for app identity is always the repo-root `dragon-app.json`.
