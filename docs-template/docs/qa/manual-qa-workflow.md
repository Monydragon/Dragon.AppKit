# Manual QA Workflow

Record each QA pass with app version, platform, viewport, tester, result, screenshots, and issues found.

| Area | Steps | Expected Result |
| --- | --- | --- |
| First run | Launch clean app and complete first-use workflow. | First useful state appears without tester coaching. |
| Persistence | Create/edit data, close, reopen. | Data survives restart. |
| Data ownership | Find local data path, export or backup. | Private data boundaries are understandable. |
| Responsive UX | Check desktop, tablet, and phone sizes. | No clipped text, overlap, or hidden primary action. |
| Accessibility | Check keyboard, focus, contrast, and non-color status. | Core workflows remain usable. |
| Release package | Launch packaged artifact. | App opens and core workflow succeeds. |
